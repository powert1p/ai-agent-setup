# Протокол автономности — для ВСЕХ агентов

## Главное правило
НИКОГДА не спрашивай пользователя ТЕХНИЧЕСКИЕ вопросы.
Не пиши "should I proceed?" — PROCEED.
Не пиши "is this approach OK?" — план утверждён, ДЕЛАЙ.

### МОЖНО спрашивать (через coordinator → user)
- Бизнес-логика которой нет в docs/product.md ("как вы считаете retention?")
- Приоритеты ("делать A или B первым?")
- Доменные знания ("что значит конкретный бизнес-термин?")
- **brainstorming skill**: задаёт бизнес-вопросы по дизайну. Это НЕ нарушение автономности — это обязательная часть workflow

### НЕЛЬЗЯ спрашивать
- Куда класть файл → читай docs/module-map.md
- Какой паттерн → читай существующий код через Grep/Glob
- Как исправить баг → читай error message, ищи в коде, потом в интернете
- Одобрение плана → plan reviewer одобряет, не человек. **НО**: brainstorming skill задаёт вопросы = РАЗРЕШЕНО

## Decision Tree

### РЕШАЙ САМ
- Архитектура: выбор между 2 подходами → бери проще
- Тип/интерфейс: не знаешь → читай backend schema
- Файл: куда класть → читай module-map.md
- Import: какой путь → grep по существующему коду
- Тест: что тестировать → happy path + error + edge case

### ЭСКАЛИРУЙ КООРДИНАТОРУ (message coordinator)
- Спека противоречит существующему коду
- Нужно изменить >10 файлов не упомянутых в спеке
- 3 попытки исправить ошибку не помогли

### ЭСКАЛИРУЙ ЧЕЛОВЕКУ (coordinator → user)
- DROP TABLE / DROP COLUMN / DELETE без WHERE
- Deploy в production
- Изменение auth/security не предусмотренное спекой
- 3+ неудачных цикла на уровне coordinator'а
- Бизнес-вопрос без ответа в docs/product.md

## Error Recovery

1. Ошибка → читай error message внимательно
2. Попытка #1 → исправь → запусти тесты
3. Попытка #2 → другой подход → запусти тесты
4. **ПЕРЕД попыткой #3 → ОБЯЗАТЕЛЬНО WebSearch/context7** по ошибке. Не гадай — ищи
5. Попытка #3 → с учётом найденного → запусти тесты
6. Если всё ещё не работает → message coordinator: "Failed after 3 attempts: [error]. Tried: [что пробовал]. Found online: [что нашёл]"

## Контекстная изоляция

Ты стартуешь с НУЛЕВЫМ контекстом. Читай ТОЛЬКО:
1. Plan file или spec file (путь в задаче)
2. Файлы codebase (через Read/Grep/Glob)
3. PROJECT.md (общий контекст проекта)

НЕ полагайся на историю чата или сообщения от других агентов без верификации. Если тебе передали "API shape такой-то" — проверь в коде что это правда.

---

## Тест-протекция

НЕДОПУСТИМО (UNACCEPTABLE) модифицировать или удалять существующие тесты чтобы код прошёл.
- Тест падает → исправь КОД, НИКОГДА тест
- Единственное исключение: тест покрывает удалённый функционал (явно указано в спеке)
- Если не можешь понять почему тест падает — message coordinator, НЕ удаляй тест

---

## File Ownership

КАЖДЫЙ файл принадлежит РОВНО ОДНОМУ агенту в рамках параллельного исполнения.
- Два агента НЕ модифицируют один файл одновременно
- Если два изменения в одном файле → объедини в одну задачу ИЛИ выполняй последовательно
- Plan файл определяет задачи и файлы для каждого агента

## Definition of Done

### Code Implementation
- [ ] Код без синтаксических ошибок
- [ ] Все существующие тесты проходят
- [ ] Новые тесты написаны (happy + error + edge)
- [ ] Следует паттернам проекта (проверь module-map)
- [ ] SQL параметризованный (формат зависит от драйвера). Никаких f-string
- [ ] Нет hardcoded секретов
- [ ] Комментарии на русском

### Data Pipeline
- [ ] Миграция idempotent (IF NOT EXISTS, ON CONFLICT)
- [ ] MV: UNIQUE INDEX для REFRESH CONCURRENTLY
- [ ] SQL проверен через MCP: NOT NULL, NOT 0
- [ ] Revenue deviation <5%

### Frontend
- [ ] TypeScript типы совпадают с Pydantic schemas
- [ ] React Query (НЕ useState+useEffect для данных)
- [ ] Компонент <150 строк, Tailwind only
- [ ] `npx tsc --noEmit` проходит
- [ ] Playwright верификация: snapshot + console + screenshot

### Test
- [ ] Happy path
- [ ] Error case (invalid input, 404, 403)
- [ ] Edge case (null, empty, 0)
- [ ] Тесты проходят (команда из PROJECT.md)
- [ ] Нет sleep/setTimeout в тестах

## Коммуникация

- **Message coordinator**: когда задача выполнена, заблокирован, нужно решение
- **Message teammate**: когда передаёшь артефакт (API shape, types, test results)
- **НИКОГДА message human напрямую** — всё через coordinator
- Формат сообщений: конкретные факты, не мнения. Пиши file:line, SQL результаты, error output
